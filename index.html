Sequenced design with 1000+ transistors

  • Random numbers are pseudoe-generated using Linear-Feedback-Shift-Register circuits that use more than a 1000 transistors
  • The sampling is performed sequentially, limited by the clock speeds. Faster hardware is still serial.

Sequencerless design with 3 transistors

  • Naturally probabilistic bits (p-bits) eliminate the need to use 1000+ transistors. Each p-bit has a naturally stochastic digital output that fluctuates between 0 and 1 at a rapid.
  • The autonomous architecture allows p-bits to flip freely, unconstrained by clock speeds, allowing massive parallelism in sampling

Cheaper, faster, efficient

  • Cheaper: Silicon footprint reduction from 1000+ transistors to just 3 with no expensive brute-force transistor scaling
  • Faster: Each p-bit can fluctuate every 100ps, which is amplified many folds owing to the massive parallelism affored by the clockless autonomous design
  • Efficient: A 1Mb network can operate at a rate of 10 Peta-flips per second, consuming 10W power

The p-computer can be implemented using existing Semiconductor and Spintronics technology, tools and materials

Higher speeds at low power, from mobile phones to supercomputers

  • Probability is ubiquitous in the world of computing. Any computation that involves uncertainty, random variables, stochastic algorithms, statistical simulations, Monte Carlo, or sampling algorithms is probabilistic by nature
  • Equally ubiquitously, today, these computations are performed by deterministic logic bits, i.e., 0’s and 1’s. However, probabilistic processors can drastically outperform deterministic processors on probabilistic computing
  • Probabilistic co-processors can function at high speed and low power for machine learnig, drug design, risk management, business analytics, ...

FPGA → ASIC → Ultimate ASIC

Performance and power consumption can be improved in several stages. The first stage starts with a software platform using Field Progammable Gate Arrays (FPGAs) alreday available at Amazon AWS and Microsoft Azure.

  • Build the software platform using FGPA. FPGA's have been specifically programmed to emulate the P-Computer
  • The FPGA implementation is competetive with the state of the art (see Purdue-P in the table)
  • Build the hardware platform based on an ASIC using existing semiconductor foundry process design kit
  • Advance the hardware platform based on the ultimate ASIC by adapting the already available MRAM technology (see Projected in the table)
